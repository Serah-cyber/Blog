ICMP Deep Dive: Understanding the Internet Control Message Protocol

16 November 2025·762 words·4 mins·
Author
SERAHER

Introduction to ICMP
#

The Internet Control Message Protocol (ICMP) is a core component of the Internet Protocol (IP) suite. While often overlooked, it plays a vital, non-data transfer role in network operations.

What ICMP Is and Its Primary Purpose
#

ICMP acts as the error reporting and diagnostic mechanism for the IP layer. It is NOT used to exchange data between systems (like TCP or UDP), but rather to communicate information about network issues.

Common utilities that use ICMP:

  • ping: Uses Echo Request (Type 8) and Echo Reply (Type 0).
  • traceroute: Uses Time Exceeded (Type 11) messages.

How an ICMP Message is Transmitted and Captured
#

ICMP messages are encapsulated directly within an IP packet.

Transmission
#

  1. Trigger: An event occurs (e.g., a host is unreachable, or a ping command is issued).
  2. Encapsulation: The ICMP message (containing its Type, Code, and Checksum) is placed into the Data/Payload section of a standard IP packet.
  3. Delivery: The IP packet is routed across the network like any other packet.

Capture
#

ICMP packets are easily captured using network analysis tools like Wireshark. Since they travel across the network, they are visible to any device in the path or any sniffing tool.

What Happens In Transit?
#

Time-to-Live (TTL)
#

ICMP is integral to preventing routing loops. Every router decrements the TTL field in the IP header.

When a router receives a packet and the TTL is decremented to zero, the router discards the packet and sends an ICMP Type 11 (Time Exceeded) message back to the source. This is how traceroute works!

Unreachable Destinations
#

If a router determines a host or network is unreachable (e.g., an incorrect route exists), it sends an ICMP Type 3 (Destination Unreachable) message back to the source host.

Analyzing an ICMP Packet (Security Perspective)
#

Analyzing the ICMP packet header is crucial for security monitoring.

Field Security Relevance
Type The most important field. Type 8 (Echo Request) is Ping. Other types can indicate scanning or malicious activity.
Code Provides specifics for the Type (e.g., Code 13 for Type 3 indicates “Communication Administratively Filtered”).
Data/Payload The payload of a Ping is often filled with random data, but it can be used for ICMP Tunneling to covertly exfiltrate data.

Detecting ICMP Tunneling
#

Defensive analysis should look for unusually large ICMP packets or ICMP packets with an unusual frequency or a payload that looks like encoded data.

Wireshark ICMP Analysis
#

This section explains how to capture, view, inspect, and interpret ICMP packets using Wireshark.
#

Step 1 — Start Wireshark & Select Your Interface
#

  1. Open Wireshark.
  2. Choose your network interface (usually eth0 or wlan0).
  3. Click Start Capturing.

#

Step 2 — Apply a Basic ICMP Filter
#

In the filter bar, type:

icmp

Press Enter.

You should now only see ICMP packets.

Step 3 — Generate ICMP Traffic (Ping)

Run:

  • Echo Request (Type 8)
  • Echo Reply (Type 0)

Step 5 — Observing the ICMP Flow
#

Ping produces a predictable request/response pattern:

  • Source → Destination (Echo Request)
  • Destination → Source (Echo Reply)

The IP header shows:

  • TTL decreasing as packets travel
  • Source and destination IP addresses
  • Packet length
  • No fragmentation unless payload is large

Step 6 — Capturing ICMP Time Exceeded (Type 11)
#

To observe TTL expiration, run:

traceroute 8.8.8.8

Now apply the filter:

icmp.type == 11

You should see Time Exceeded packets returned by intermediate routers.

If you see no results, your ISP or network may block traceroute packets. Try:

traceroute -I 8.8.8.8

(Uses ICMP instead of UDP)

Step 7 — Detecting Large ICMP Packets (Tunneling)
#

To check for suspicious packets:

icmp && frame.len > 200

If nothing shows up → good, your network has no large ICMP payloads.

Conclusion
#

ICMP is a necessity for network health, but its simple nature makes it a target for diagnostic tools and covert channels. Understanding its types and transit behavior is essential for any Blue Team operation.
